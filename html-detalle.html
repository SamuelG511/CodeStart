¿Cómo se construyó CodeStart? (Guía Técnica)

Para que esta página pasara de un bloc de notas a una web "Plus", seguimos una arquitectura de tres niveles:

Nivel 1: El Esqueleto Semántico

No usamos etiquetas al azar. La estructura real es esta:

  • nav: El contenedor del menú. Vital para la navegación.
  • <header>: Donde vive el título impactante. Solo hay uno por página.
  • <main>: La zona donde ocurre el contenido principal.
  • <section>: Los bloques que dividen el contenido por tema.
  • <footer>: Información de cierre y navegación secundaria.
<!-- Estructura semántica de CodeStart -->
<nav class="menu-navegacion">...</nav>
<header class="cabecera-principal">...</header>
<main class="contenedor-web">
    <section class="tarjeta-blanca">...</section>
    <section class="seccion-gris">...</section>
</main>
<footer class="pie-pagina">...</footer>

Nivel 2: El Sistema de Clases (CSS)

El secreto profesional fue reutilizar. En lugar de darle estilo a cada elemento, creamos moldes:

/* El molde de las tarjetas blancas */
.tarjeta-blanca {
    background: white;
    padding: 40px;
    border-radius: 12px;
    box-shadow: 0 5px 25px rgba(0,0,0,0.05);
    margin-bottom: 30px;
}
  • background: Color de fondo con keyword "white".
  • padding: Relleno interno con shorthand de 1 valor.
  • border-radius: Esquinas redondeadas.
  • box-shadow: Sombra con rgba para profundidad sutil.

¿Por qué así? Porque con class="tarjeta-blanca" obtienes diseño instantáneo en cualquier sección nueva.

Nivel 3: El Contenedor Maestro

Todo el contenido vive dentro de .contenedor-web:

.contenedor-web {
    max-width: 1100px;       /* Limita el ancho máximo */
    margin: -40px auto 60px; /* Centrado + margen negativo */
    padding: 0 20px;         /* Espacio lateral en móviles */
}
  • max-width: Evita que el contenido se estire en pantallas grandes.
  • margin: auto: Centra el bloque horizontalmente.
  • margin negativo: Sube el contenido sobre la cabecera para el efecto visual.

Resumen para el Programador

Hicimos esta página separando el Contenido (HTML) del Estilo (CSS). Si mañana quieres cambiar el color del logo, cambias una línea en el CSS y toda la web se actualiza. Eso es eficiencia real.

Separación de Responsabilidades

HTML = estructura. CSS = presentación. Nunca mezclados en el mismo archivo.

Clases Reutilizables

Un sistema de clases bien diseñado escala sin duplicar código.

Hojas Externas

Un solo style.css para todas las páginas. El navegador lo cachea y carga más rápido.

La diferencia entre un sitio web amateur y uno profesional no está en la complejidad — está en la organización y la consistencia.
